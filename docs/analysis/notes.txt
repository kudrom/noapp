Registry.cpp
    + The Registry is good.
    + What's the protocol to create/register/initialize a new Agent into the Registry? -> It's all in the CreateDialogTree diagram, to sum up:
       +.1 first it's called Create by the Registry, who creates its concepts, calls its subagents and OnCreation callback.
       +.2 then it's called Initialize by the DTTManager, who callback OnInitialization with the groundingModel already instantiated, this callback is also called by Reset
       +.3 then it's called Register by the GTTManager, that initializes the TriggeredConcepts
       +.+ All callbacks can be overwritten by the RCTSL

Agent.cpp

    + The configuration of an agent is too much C-like, we should use a common class and provide an abstraction of the different config files in the FS that translates to a volatile config for every CoreAgent.
